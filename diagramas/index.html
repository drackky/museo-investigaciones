💡 ¿Qué es un Microservicio?

En lugar de escribir todo el código en un único programa gigante (llamado monolito), dividimos el sistema en programas pequeños e independientes llamados microservicios. Cada uno tiene una única responsabilidad:

  • Auth Service: Únicamente registra usuarios y gestiona contraseñas.
  • Documents Service: Solo manipula metadatos y etiquetas de archivos.
  • Collections Service: Organiza grupos o carpetas de documentos.

🔑 ¿Por qué usamos un API Gateway?

El API Gateway es el vigilante en la puerta de entrada. En lugar de que el Frontend tenga que recordar en qué puerto corre cada servicio (5001, 5002, etc.), el Frontend solo le hace peticiones al Gateway (puerto 5000). El Gateway se encarga de:

  • Ruteo Inteligente: Desvía la petición al servicio interno correspondiente.
  • Seguridad Unificada: Evita problemas con políticas de CORS del navegador.
  • Filtro de Seguridad: Oculta la red interna de microservicios al exterior.
